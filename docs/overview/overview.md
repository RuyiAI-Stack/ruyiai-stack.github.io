# 总览

# 建设背景

人工智能基础设施正加速进入以推理为中心的发展阶段。以大语言模型为代表的人工智能模型，正在云、边、端等多种场景中快速落地，对算力平台的适配效率、部署效率和可编程能力提出了更高要求。RISC-V 具备开放、模块化和可扩展的指令集架构特点，能够较好支撑人工智能计算对定制指令集、软硬件协同设计和生态持续演进的需求。近年来，RISC-V 硬件能力持续提升，正在成为人工智能计算平台的重要发展方向。

与此同时，RISC-V 平台在人工智能部署中仍面临明显的软件碎片化挑战。不同硬件平台在 Vector、Matrix 及异构加速等方向上存在多样化实现方式，使得人工智能模型部署与优化往往依赖面向具体平台的手工移植和调优。当模型结构或硬件平台发生变化时，现有方法通常需要重新设计编译与优化流程，导致适配周期长、部署效率低，难以满足人工智能模型与硬件平台快速演进的需求。因此，面向 RISC-V 构建统一、高效、可扩展的系统软件栈，已成为支撑人工智能应用落地的重要基础。

# 建设内容

RuyiAI 系统软件栈面向 RISC-V 硬件平台，致力于构建统一的人工智能系统软件基础能力。一方面，RuyiAI 坚持“Ruyi 原生”路线，以 RISC-V 为主要目标架构开展系统化设计与优化，形成面向人工智能计算的原生软件栈；另一方面，RuyiAI 也支持“Ruyi 适配”路线，面向主流人工智能软件生态，提供向 RISC-V 平台高效迁移、适配与运行的能力，从而支撑人工智能应用在不同场景下的持续落地与演进。

![RuyiAI 系统软件栈建设内容](/docs/overview/overview-1.png)

在总体架构上，RuyiAI 以主流人工智能开发生态和开放编译基础设施为核心支撑。围绕 PyTorch 与 LLVM/MLIR 构建面向 RISC-V 的原生 AI 软件栈，形成覆盖模型接入、图与张量表示、多级编译优化、编程语言、算子库、运行时与统一硬件抽象的系统化能力，从而实现人工智能模型在 RISC-V 平台上的高效部署、运行与持续演进。

# 建设目标

RuyiAI 系统软件栈以真实应用需求为牵引，面向人工智能模型在 RISC-V 平台上的部署、运行与优化过程，形成持续迭代、持续验证的建设机制。通过紧贴实际应用场景与典型计算负载，不断推动系统软件能力与硬件平台能力协同演进，提升人工智能模型从适配到落地的整体效率。

RuyiAI 将持续完善推理框架、AI 编译器、AI 算子库、AI 编程语言与运行时系统等核心组成部分，构建覆盖主流 RISC-V 硬件平台与典型 AI 应用场景的系统软件能力体系，逐步形成统一、高效、可扩展的人工智能系统软件生态。
