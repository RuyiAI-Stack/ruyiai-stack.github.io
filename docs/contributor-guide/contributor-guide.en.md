# Contributor Guidelines

Welcome to the **RuyiAI** open-source community! We welcome contributions in all forms, including code submissions, documentation improvements, test case additions, and Issue discussions.

To ensure codebase consistency and efficient collaboration, all contributors must adhere to the following principles and review the relevant documentation before submitting code or initiating a Pull Request.

## Contribution Principles

*   **Originality & Compliance**: Follow the project's open-source license and ensure contributions are original and free of copyright disputes.
*   **Code Style**: Maintain code readability and maintainability by strictly adhering to the project's established coding standards.
*   **Commit Convention**: Ensure commit messages are clear and that each commit focuses on a single modification goal.
*   **Workflow**: Follow the unified Git collaboration workflow to ensure clean branch management and commit history.
*   **Community Atmosphere**: Maintain friendly communication and rational discussions to foster an open and inclusive development environment.

## Related Guidelines and Workflow

Before submitting code, please read the following detailed documentation:

*   **[Code Style](https://ruyiai-stack.github.io/docs.html#code-style)**
*   **[Git Workflow](https://ruyiai-stack.github.io/docs.html#git-workflow)**

If you find a Bug or have a new feature suggestion, please feel free to contact us via GitHub Issues.
