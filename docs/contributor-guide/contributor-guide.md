# 贡献者指引

欢迎参与 **RuyiAI** 开源社区！我们欢迎代码提交、文档完善、测试用例补充及 Issue 讨论等各类形式的贡献。

为保证代码库的一致性与协作流程的高效，所有贡献者在提交代码或发起 Pull Request 前，请务必遵循以下原则并查阅相关文档。

## 贡献原则

*   **原创与合规**：遵循项目开源协议，保证贡献内容原创且无版权争议。
*   **代码规范**：保持代码的可读性与可维护性，严格遵守项目既定的代码风格。
*   **提交规范**：Commit 信息清晰，且每次提交应聚焦于单一修改目标。
*   **协作流程**：遵循统一的 Git 协作流程，确保分支管理与提交历史整洁。
*   **社区氛围**：保持友善沟通与理性讨论，共同维护开放包容的开发环境。

## 相关规范与流程

提交代码前，请务必阅读以下详细文档：

*   **[代码规范约束](https://ruyiai-stack.github.io/docs.html#code-style)**
*   **[Git 协作工作流程](https://ruyiai-stack.github.io/docs.html#git-workflow)**

如果您发现了 Bug 或有新的功能建议，欢迎通过 GitHub Issues 与我们联系。
