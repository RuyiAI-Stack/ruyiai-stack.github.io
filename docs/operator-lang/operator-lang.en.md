# RuyiAI Operator Programming Language

![RuyiAI Operator Programming Language](/docs/operator-lang/operator-lang.png)

To meet high-performance operator development needs, RuyiAI supports operator programming frameworks based on **Triton** and **TileLang**, and builds **Triton-RISCV** and **TileLang-RISCV** for the RISC-V platform. It provides a unified operator development entry point and a compilation/execution infrastructure. With Python-based interfaces, developers can describe computation logic in high-level operator languages, improving productivity while retaining strong expressiveness and optimization space for specific hardware platforms.

In the compilation pipeline, RuyiAI normalizes language-level IR generated by different frontends into the **MLIR Linalg** abstraction level, enabling unified representation and converged compilation paths at the IR layer. On top of this, the system further relies on **Buddy Compiler** to continuously lower, optimize, and generate target code from Linalg-level MLIR, and finally achieves efficient execution on RISC-V platforms. Through this technical path of "operator programming language -> unified intermediate representation -> lowering -> deployment", RuyiAI provides unified, open, and extensible system support for multi-language operator development.
